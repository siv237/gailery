# КРИТИЧНЫЕ ПРАВИЛА (НАРУШЕНИЕ = БАГ)

1. **НИКОГДА не делай git commit/push без прямого распоряжения пользователя.** Жди команду «комить», «пуш», «запуши».
2. **Перед ЛЮБЫМ действием описывай словами что и зачем делаешь.** Никаких немых правок.
3. **Не обрезать вывод команд.** Пользователь должен видеть полный вывод.
4. **Не downgrade пакеты без согласия.**
5. **Останавливать pipeline только кнопкой стоп (флаг no_restart), не kill процессов вручную.**
6. **Удалять .pyc из __pycache__ после изменений Python файлов.**
7. **Тесты только на тестовом окружении. Никаких реальных действий с системой из тестов.**
8. **Работать на русском языке.**
