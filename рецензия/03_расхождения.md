## Детальное описание ключевых расхождений

### 1. Порядок шагов

```
Идеальный:  SCAN → EXIF → FACES → DESCRIBE → EMBED
                   ─────   ─────   ────────
                   без GPU  GPU     GPU
                            быстро  медленно

Текущий:    SCAN → DESCRIBE → FACES → EXIF → EMBED
                   ────────   ─────   ─────
                   GPU         GPU    без GPU
                   медленно    быстро
```

**Проблема текущего порядка:**
- describe работает без информации о лицах → описания без имён
- faces зависит от `faces_present` из describe → циклическая зависимость
- EXIF после GPU-шагов → пользователь ждёт часы до появления дат в UI

### 2. faces.py — зависимость от describe

```
Текущий:   describe ставит faces_present=1 → faces обрабатывает только эти фото
Идеальный: faces сканирует ВСЕ фото → ставит faces_present сам (ground truth)
```

InsightFace: ~50-100мс/фото. Даже для 10k фото — 17 минут. Нет смысла предварительно фильтровать через медленный llama (~3с/фото).

### 3. describe.py — без контекста лиц

```
Текущий промпт:    [фото] → «Двое людей за столом»
Идеальный промпт:  [фото] + [Мария слева, Иван справа] → «Мария и Иван за столом»
```

Для этого describe.py должен:
- Запросить faces + personas для текущего content_hash
- Сформировать контекст (имена, позиции из bbox)
- Добавить в system/user prompt

### 4. Каскад при изменении лиц — неполный

| Операция | Текущее поведение | Нужное поведение |
|---|---|---|
| rename (display_name) | Ничего не происходит | re-describe + re-embed |
| merge | re-embed только | re-describe + re-embed |
| delete | Ничего | re-describe + re-embed |
| assign (лицо другой персоне) | Ничего | re-describe + re-embed |

При новом порядке описания содержат имена → имя изменилось → описание устарело → embed тоже.

### 5-6. SQLite-арбитраж и хеширование

- MQTT-механизм `db/cmd` → pipeline.py существует и работает, но нигде не документирован
- Хеширование инлайн при скане, а не отдельной фазой — PIPELINE.md врёт

### 7-8. Видео и расширения

- exif.py фильтрует `media_type != 'video'` — видео не получают метаданные
- Три набора видео-расширений несогласованы → некоторые форматы не попадают в каталог

### 9-11. Документация

Мелкие расхождения между PIPELINE.md и кодом: неправильный воркер наполнения, несуществующий флаг hash, неточные формулы счётчиков.
