## Этап 1: БЛОКЕРЫ — новый порядок шагов

> Без этих изменений нельзя переставить FACES до DESCRIBE

### 1.1 faces.py — убрать зависимость от faces_present
- **Файл:** `faces.py`
- **Что:** Выборка фото: `faces_done=0` вместо `faces_present=1`
- **Логика после обработки:**
  - Лица найдены → `faces_present=1, faces_done=1`
  - Лиц нет → `faces_present=0, faces_done=1`
- **Добавить колонку:** `faces_done` в photos и catalog_files (если нет)
- **Трудоёмкость:** 2-4 часа (логика + миграция БД + тесты)
- **Тест:** Запустить faces.py на фото без описания — должен работать

### 1.2 describe.py — передать имена лиц в промпт
- **Файл:** `describe.py`
- **Что:**
  1. Для каждого фото запросить faces + personas (по content_hash)
  2. Сформировать контекст: имена, позиции (bbox → «слева/справа/центр»)
  3. Добавить в промпт llama:
  ```
  На фото обнаружено 2 лица:
  - Мария (слева, 30% ширины кадра)
  - Иван (справа, 25% ширины кадра)
  Опиши что на фото, используя имена людей.
  ```
  4. Если лиц нет или персоны не именованы — промпт без контекста (как сейчас)
- **Трудоёмкость:** 3-5 часов
- **Тест:** Описать фото с именованными лицами — описание должно содержать имена

### 1.3 pipeline.py — переставить порядок шагов
- **Файл:** `pipeline.py`, main()
- **Что:** Изменить порядок вызовов:
  ```python
  # Было: SCAN → DESCRIBE → FACES → EXIF → EMBED
  # Стало:
  run_step("QUICK SCAN", scan_args)        # 1. НАПОЛНЕНИЕ
  run_step("EXIF", exif_args)              # 2. EXIF
  run_step("FACES", faces_args)            # 3. ЛИЦА
  # video descriptions '[видео]'            # перед describe
  run_step("DESCRIBE", describe_args)      # 4. ОПИСАНИЕ
  run_step("EMBED", embed_args)            # 5. ИНДЕКСАЦИЯ
  ```
- **Трудоёмкость:** 30 минут
- **Тест:** Запустить pipeline, убедиться что шаги идут в правильном порядке

### 1.4 Каскад при изменении персоны
- **Файлы:** `src/api/persons.py`, `src/database.py`, `pipeline.py (_execute_db_cmd)`
- **Что:** Создать функцию `invalidate_for_persona(persona_id)`:
  ```python
  def invalidate_for_persona(persona_id):
      """Сброс описания + эмбеддинга для всех фото с лицами этой персоны"""
      hashes = db.get_content_hashes_for_persona(persona_id)
      for ch in hashes:
          # сброс описания
          db.sqlite.execute("UPDATE photos SET description=NULL WHERE path IN "
              "(SELECT abs_path FROM catalog_files WHERE content_hash=? AND is_canonical=1)", (ch,))
          db.sqlite.execute("UPDATE catalog_files SET described=0 WHERE content_hash=? AND is_canonical=1", (ch,))
          # сброс эмбеддинга
          db.sqlite.execute("UPDATE photos SET embedded=0 WHERE path IN "
              "(SELECT abs_path FROM catalog_files WHERE content_hash=? AND is_canonical=1)", (ch,))
          db.sqlite.execute("UPDATE catalog_files SET embedded=0 WHERE content_hash=? AND is_canonical=1", (ch,))
          # удаление из LanceDB
          photo = db.get_photo_by_content_hash(ch)
          if photo: db.delete_photo_embedding(photo["photo_id"])
      db.sqlite.commit()
  ```
- **Вызывать из:**
  - `update_persona` (rename) — **сейчас не вызывается!**
  - `merge_personas` — добавить сброс description (сейчас только embed)
  - `delete_persona` — добавить
  - `reassign_face` — добавить (если есть)
- **В pipeline.py `_execute_db_cmd`:** добавить обработку команды `invalidate_persona`
- **Трудоёмкость:** 2-3 часа
- **Тест:** Переименовать персону → убедиться что description сбросился, pipeline переопишет
